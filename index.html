FAQ

Everything You Need to Know

Questions about security, permissions, and how it all works.

Is Cloud Thrift secure? What access does it need to my AWS account?
+
Cloud Thrift connects via a read-only IAM role by default — we cannot make any changes to your infrastructure unless you explicitly enable auto-fix mode. All data is encrypted in transit and at rest. We are SOC 2 Type II certified and GDPR compliant. No credentials are ever stored on our servers.
What IAM permissions does Cloud Thrift require?
+
For read-only analysis: ReadOnlyAccess managed policy is sufficient. For auto-optimization, we provide a minimal custom policy that grants only the specific permissions needed: ec2:StopInstances, ec2:DeleteVolume (with conditions), rds:StopDBInstance, etc. We publish our exact IAM policy JSON on GitHub so you can audit every permission before granting it. The principle of least privilege is baked into our design.
Can I roll back changes made by Cloud Thrift?
+
Yes. Every action Cloud Thrift takes is logged with full audit trail. For destructive actions (like deleting EBS volumes), we always create a backup first. You can roll back any action within 30 days. For instance stops, just start them again — Cloud Thrift never terminates instances without explicit approval.
How is the ~30% savings figure calculated?
+
The 31% average is based on aggregate data across 2,400+ connected AWS accounts over 12 months. Individual results vary based on your workload type, how much you've already optimized, and which features you enable. Most teams see 15–45% reduction. We show you a projected savings estimate during onboarding before any changes are made.
What support options are available?
+
Free tier gets community forum support. Pro plan includes priority email support with 24-hour response SLA. Enterprise customers get a dedicated support engineer, Slack Connect channel, and quarterly cost optimization reviews. All plans have access to our documentation, API reference, and status page.
